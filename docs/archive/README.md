here for reference only. Current implementation and operations guidance belongs
in:

- [`../../README.md`](../../README.md)
- [`../../CLAUDE.md`](../../CLAUDE.md)
- [`../README.md`](../README.md)
- [`../ARCHITECTURE.md`](../ARCHITECTURE.md)

Do not use archived D1-era plans or specs as current implementation guidance.
When an archive note conflicts with a current doc, the current doc wins.

## Map

- `plans/` - dated implementation plans that have already been executed or
  superseded.
- `specs/` - design notes for completed feature work.
- `r2-json-migration-design-note-2026-05-02.md` and
  `post-r2-architecture-cleanup-2026-05-02.md` - historical context for the R2
  migration; use `../ARCHITECTURE.md` for the active model.
- `code-quality-*.md`, `*-audit-*.md`, and `refactor-candidates.md` - review
  snapshots retained to explain past cleanup decisions.
- `todo-2026-05-02.md` - previous TODO snapshot; use `../TODO.md` for active
  work.
